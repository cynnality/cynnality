the 'why'

i want to learn how to create a website and host it.

rules: everything has to be done independently and for free
(NO paid platforms or services that automate the process)
(NO prebuilt templates, automation tools, or website generators)

project #1 = building this website /// project #2 = figuring out how to host it


project #1 - the website

i don't have any coding experience so im learning as i go!

im building this site from scratch with HTML, CSS and JavaScript. the idea is for it to be a place for my own personal archive and categorization of online things. basically my version of a blog.

the content & design

instead of worrying too much about how things "should be done", im treating this more as a practice in creative expression. so im only learning how to code and implement things that i want to see. this way the learning to code part comes more naturally, almost like a side effect of making something i think is cool.


project #2 - hosting and deploying website

i hope to host this on my own very soon!!!

right now, im using glitch.com to host this site for FREE!

there are a TON of options when it comes to hosting but my main goal for this part is to get a better understanding of what actually goes into deploying a website over the internet for others to access and how a website itself is accessed and pulled. im not trying to find the most efficient or the most common method, im trying to develop a sense of what is required to make it all happen.
